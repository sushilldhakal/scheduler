# Upgrade guide

## v0.4.0 (current)

### Breaking changes

**`Block.breakStartH` / `Block.breakEndH`** — new optional fields on the `Block` type. Existing blocks without these fields are unaffected. If you have custom TypeScript types that extend `Block`, you may need to add these as optional fields.

### New features

- **2D free drag** — blocks now lift out of their row and follow the cursor freely in 2D. The old cursor ghost div has been removed; the real block element moves via DOM mutation for zero layout reflows.
- **Right-click context menu** — powered by `@radix-ui/react-context-menu`. Add it to your peer dependencies if installing from source.
- **Hover popover** — fixed-position portal that escapes `contain:paint` clipping. Shows employee, category, time range, duration, break, conflict warning.
- **Row modes** — `"category"` (default, stacks shifts) and `"individual"` (one row per employee). Switch in Settings panel.
- **Resizable sidebar** — powered by `react-resizable-panels`. Add to peer dependencies if installing from source.
- **Vertical edge-scroll** — dragging near top/bottom now scrolls vertically.
- **4px desktop drag threshold** — prevents accidental block moves on click.
- **`scrollbarGutter: stable`** — header and grid are now pixel-perfect in sync regardless of vertical scrollbar presence.

### New peer dependencies

```bash
npm install react-resizable-panels @radix-ui/react-context-menu
```

### Migration

No migration steps required for existing code. All new features are additive. The `Block` type has two new optional fields — existing data is backward compatible.

## v0.3.x → v0.3.3

Patch releases only. No breaking changes. Bug fixes to:
- `hourBg()` CSS variable (extra closing paren causing transparent non-working hours)
- ListView day mode "No Shifts" display
- Sidebar literal `\n` text node rendering

## v0.2.x → v0.3.0

### Renamed props

| Old | New |
|-----|-----|
| `onShiftMove` | `onBlockMoved` |
| `onShiftCreate` | `onBlockCreate` |
| `onShiftDelete` | `onBlockDelete` |

### Breaking change: `visibleFrom` / `visibleTo`

These moved from top-level `Scheduler` props into `SchedulerConfig`. Update:

```tsx
// Before (v0.2.x)
<Scheduler visibleFrom={7} visibleTo={22} />

// After (v0.3.0+)
<Scheduler config={createSchedulerConfig({ visibleFrom: 7, visibleTo: 22 })} />
```
