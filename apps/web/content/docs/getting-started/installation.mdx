# Installation

shadcn-scheduler gives you **two installation paths**. The key difference is ownership.

## Option 1 — shadcn registry (recommended — you own the code)

```bash
npx shadcn@latest add https://shadcn-scheduler.vercel.app/registry.json
```

This copies **all 56 source files** directly into `components/scheduler/` in your project:

```
components/scheduler/
  index.ts                          ← import from here
  core/
    Scheduler.tsx                   ← the main component
    GridView.tsx                    ← the grid engine (~3500 lines)
    context.tsx
    types.ts
    constants.ts
    config.ts
    hooks/
      useAuditTrail.ts
      useFlatRows.ts
      useMediaQuery.ts
      useScrollToNow.ts
    components/
      modals/ShiftModal.tsx
      modals/AddShiftModal.tsx
      views/DayWeekViews.tsx
      views/MonthView.tsx
      views/YearView.tsx
      views/ListView.tsx
      views/TimelineView.tsx
      ui/context-menu.tsx
      ui/resizable.tsx
      settings/SchedulerSettings.tsx
      ...
  domains/
    default/index.tsx               ← roster preset
    tv/index.tsx
    healthcare/index.tsx
    ...
```

You import from your own components folder:

```tsx
import { Scheduler } from '@/components/scheduler'
```

**Why this matters:** You can edit `GridView.tsx`, add fields to `types.ts`, change the drag behaviour, add a new view — anything you want. No fork, no patch file, no fighting a third-party package.

## Option 2 — npm package

If you prefer not to own the source, use the npm package:

```bash
npm install @sushill/shadcn-scheduler
# pnpm
pnpm add @sushill/shadcn-scheduler
# yarn
yarn add @sushill/shadcn-scheduler
```

```tsx
import { Scheduler } from '@sushill/shadcn-scheduler'
```

This is simpler to keep updated (`npm update`) but you can't modify the internals.

## Requirements

- React 18+
- Tailwind CSS 3 or 4
- shadcn/ui CSS variables (or add them manually — see below)

## Peer dependencies

Both install paths need these packages:

```bash
npm install @tanstack/react-virtual react-resizable-panels @radix-ui/react-context-menu @radix-ui/react-popover lucide-react react-day-picker
```

| Package | Version | Why |
|---------|---------|-----|
| `@tanstack/react-virtual` | ≥3 | Row virtualisation (200+ staff) |
| `react-resizable-panels` | ≥2 | Resizable sidebar |
| `@radix-ui/react-context-menu` | ≥1 | Right-click menu |
| `@radix-ui/react-popover` | ≥1 | Hover popover |
| `react-day-picker` | ≥8 | Date picker in modals |
| `lucide-react` | ≥0.260 | Icons |
| `class-variance-authority` | ≥0.7 | Variant classes |
| `clsx` + `tailwind-merge` | ≥2 | Class utilities |

## Tailwind configuration

Add the scheduler to your Tailwind `content` array:

```ts
// tailwind.config.ts
export default {
  content: [
    './app/**/*.{ts,tsx}',
    './components/**/*.{ts,tsx}',
    // If using npm package, also add:
    './node_modules/@sushill/shadcn-scheduler/dist/**/*.{js,mjs}',
  ],
}
```

## CSS variables

The scheduler reads standard shadcn/ui CSS variables. If you're already on shadcn/ui — nothing extra needed.

If you're not on shadcn/ui, add these to your global CSS:

```css
:root {
  --background: 0 0% 100%;
  --foreground: 240 10% 3.9%;
  --muted: 240 4.8% 95.9%;
  --muted-foreground: 240 3.7% 15.9%;
  --border: 240 5.9% 90%;
  --primary: 240 5.9% 10%;
  --primary-foreground: 0 0% 98%;
  --accent: 240 4.8% 95.9%;

  --popover-foreground: 240 10% 3.9%;

import { Scheduler } from '@sushill/shadcn-scheduler'

export default function Test() {

    <div className="h-[500px]">
      <Scheduler
        categories={[{ id: 'c1', name: 'Team', kind: 'category', colorIdx: 0 }]}
        employees={[{ id: 'e1', name: 'Alex', kind: 'employee', categoryId: 'c1', colorIdx: 0 }]}
        shifts={[]}
        onShiftsChange={() => {}}

    </div>
  )
}
```

If you see a weekly grid — you're set.
