# Large roster (200+ staff)

The scheduler uses TanStack Virtual to render only the rows visible in the viewport. You can have 200+ employees with no performance degradation — the DOM stays small regardless of team size.

[→ Try it live](/demo)

## Recommended setup for large teams

```tsx
import { createRosterConfig } from '@sushill/shadcn-scheduler'

const config = createSchedulerConfig({ preset: 'roster', {
  initialScrollToNow: true,
  // Individual mode: one row per employee — clearest view at scale
  // Switch in the Settings panel, or set as default via rowMode prop
} })

<Scheduler
  categories={bigCategories}    // 20 departments
  employees={bigEmployees}      // 200 staff
  shifts={shifts}               // thousands of shifts
  onShiftsChange={setShifts}
  config={config}
  initialView="week"
/>
```

## Row modes

For large teams, **individual mode** (one row per employee, categories as collapsible headers) is clearer than the default category mode where all shifts stack in one row:

- Open the **Settings** panel (gear icon in toolbar)
- Switch to **Individual** mode
- Categories collapse to headers, each employee gets their own row

## Performance tips

- Use `onVisibleRangeChange` to only load shifts for the visible date range from your API
- Keep `shifts` filtered to visible employees using `selEmps` state
- The virtualizer handles row count automatically — no `windowing` or `overscan` config needed
