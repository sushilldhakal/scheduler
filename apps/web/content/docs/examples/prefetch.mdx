# Data prefetching

For large datasets, load shifts on demand as the user scrolls to new periods rather than fetching everything upfront.

[→ Try it live](/demo)

## Basic setup

```tsx
const [shifts, setShifts] = useState<Block[]>([])

const handleVisibleRangeChange = async (start: Date, end: Date) => {
  // Called when the user scrolls to a new visible range
  const newShifts = await fetchShifts({
    from: start.toISOString(),
    to: end.toISOString(),
  })
  
  // Merge with existing shifts to avoid clearing previously loaded ones
  setShifts((prev) => {
    const existingIds = new Set(newShifts.map(s => s.id))
    const kept = prev.filter(s => !existingIds.has(s.id))
    return [...kept, ...newShifts]
  })
}

<Scheduler
  shifts={shifts}
  onShiftsChange={setShifts}
  onVisibleRangeChange={handleVisibleRangeChange}
  prefetchThreshold={0.7}   // Fire at 70% scroll (loads ahead of the edge)
/>
```

## Prefetch threshold

`prefetchThreshold` (0–1) controls when `onVisibleRangeChange` fires:

- `0.5` — fires when the user reaches 50% of the scroll range (early prefetch)
- `0.8` — fires at 80% (default — good balance)
- `1.0` — fires only at the very edge (late prefetch, may show blank areas)

## With loading state

```tsx
const [isLoading, setIsLoading] = useState(false)

const handleVisibleRangeChange = async (start: Date, end: Date) => {
  setIsLoading(true)
  try {
    const newShifts = await fetchShifts({ from: start, to: end })
    setShifts(prev => mergeShifts(prev, newShifts))
  } finally {
    setIsLoading(false)
  }
}

<Scheduler
  isLoading={isLoading}
  onVisibleRangeChange={handleVisibleRangeChange}
  ...
/>
```
