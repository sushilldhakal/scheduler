# Full roster demo

The **full roster** configuration showcases the primary day/week scheduling workflow.

## What this demo highlights

- Week view with dozens of staff across multiple categories.
- Drag-and-drop editing of shifts.
- Draft vs published states and bulk actions.
- Zoom controls and scroll-to-now.

## How to run locally

1. Start the docs app:

```bash
npm run dev:web
# then visit http://localhost:3002/docs
```

2. Navigate to **Examples → Full roster demo**.

3. Inspect the underlying usage in the demo page:

```tsx
<Scheduler
  categories={categories}
  employees={employees}
  shifts={shifts}
  onShiftsChange={setShifts}
  initialView="week"
/>
```

Use this as a reference when wiring the scheduler into your own roster or HR system.
