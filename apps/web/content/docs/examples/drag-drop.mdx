# Drag & drop

The scheduler supports **2D free drag** — blocks lift out of their row and follow your cursor freely across the entire grid. Drop on any row, any time slot, any day.

[→ Try it live](/demo)

## Basic usage

Drag and drop is enabled by default. No configuration required:

```tsx
<Scheduler
  categories={categories}
  employees={employees}
  shifts={shifts}
  onShiftsChange={setShifts}
/>
```

## Track moves with onBlockMoved

```tsx
<Scheduler
  onBlockMoved={(block, newDate, newStartH, newEndH) => {
    console.log(`${block.employee} moved to ${newDate} ${newStartH}:00–${newEndH}:00`)
    // Sync with your API
    updateShift(block.id, { date: newDate, startH: newStartH, endH: newEndH })
  }}
/>
```

## Disable drag on specific blocks

Set `draggable: false` on any block to make it immovable:

```tsx
const shifts = [
  {
    id: 's1',
    draggable: false,  // This block cannot be dragged
    // ...
  }
]
```

## Disable drag for all blocks

```tsx
<Scheduler
  config={createSchedulerConfig({
    badgeVariant: 'resize',  // resize only, no drag
    // or:
    badgeVariant: 'none',    // no drag or resize
  })}
/>
```

## Disable resize only

```tsx
const shifts = [
  { id: 's1', resizable: false, /* ... */ }
]
```

## Read-only mode

Disable all interactions at once:

```tsx
<Scheduler readOnly />
```

## How the drag works

1. **Desktop** — click and hold a block, move 4px to commit to drag mode (prevents accidental drags on click)
2. **Touch** — long-press (500ms) to start drag, then move freely
3. **Drop zone ghost** — snapped semi-transparent overlay shows exactly where the block will land
4. **Edge scroll** — drag near the edges to scroll horizontally (week/multiday) or vertically (large team) — accelerates the closer you get
